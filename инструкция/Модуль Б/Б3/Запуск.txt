1) Сгенерировать метки + CSV на ручную проверку
python -m agents.b2_labeling.main --k 3 --active_n 10

Появится файл:

agents/b2_labeling/artifacts/active_review.csv

Ты открываешь его, правишь колонку review_label у спорных треков.

2) Импортировать ручные метки обратно в БД
python -m agents.b2_labeling.main --import_review_csv agents/b2_labeling/artifacts/active_review.csv

таблица alerts - служебная таблица, инфа по трека 
alerts — это служебная таблица, куда агент B3 (или ваш дашборд/логика оповещений) складывает события-уведомления по трекам.

Простыми словами: это журнал “что-то важное произошло на маршруте”.

Зачем она нужна

хранить результаты проверок/правил (порог риска, скорость, резкий подъём/спуск, близость к воде, плохая погода и т.п.)

показывать “последние алерты” в интерфейсе (у тебя как раз внизу таблица “Последние алерты”)

потом можно делать фильтрацию по треку, времени, типу алерта и серьёзности

Обычно какие поля там бывают

По твоему UI видно типичный набор:

id — идентификатор алерта

track_id — к какому треку относится

track_point_id — к какой точке трека (если привязано к конкретной)

ts — время (или точки, или вычисления)

kind — тип алерта (например risk_high, speed_anomaly, flood_risk и т.д.)

severity — уровень серьёзности (low/medium/high)

value — измеренное значение (например риск 0.82 или скорость 18 км/ч)

threshold — порог, который был превышен

message — человекочитаемое описание

created_at — когда записали алерт в БД
